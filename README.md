### 一、插件简介

**联机工具** 是基于 **JS-Slash-Runner**（酒馆助手）开发的 **SillyTavern（傻酒馆）** 多人实时协作插件，支持**在线公共服务器**与**本地频道**两种模式。  

多名用户（玩家 + 观众）可同时在一个房间内：
- 共同向 AI 提交回复（输入池合并）
- 房主统一合并后发送给 AI
- 实时同步聊天记录、流式输出、正则、MVU 变量、数据库
- 支持**隐藏模式**（他人仅看到 `********`，重 roll 时自动补回系统提示）
- 支持**观众模式**（只看不提交）
- 支持**房主转让**、**限时自动发送**、**自动重连**、**剧透遮罩** 等特性

插件以**悬浮毛玻璃面板**形式呈现，支持拖拽、最小化、移动端适配，完美融入各种酒馆美化主题。

---

### 二、核心特性

#### 基础功能

- 在线模式：连接公共服务器（默认 `https://room.yufugemini.cloud`）
- 本地模式：同一浏览器/同局域网多标签通过 BroadcastChannel 通信（端口可自定义，默认 2157）
- 创建/加入带密码房间
- 在线房间列表实时刷新 + 人数/锁图标/创建者显示
- 玩家 / 房主 / 观众 三种角色动态切换
- 房主离开后**自动转让**（按加入顺序推选下一位玩家）

#### 输入协作系统

- **输入池**：实时显示所有人提交内容（房主可见全部，普通玩家可见自己 + 他人脱敏内容）
- **隐藏模式**：开启后他人看到 `********`，重 roll 时房主自动把隐藏内容作为系统提示补回，保证生成一致
- **限时自动发送**（房主专属）：有人提交后倒计时，超时自动合并发送
- 一键**撤回自己已提交输入**
- **Persona 同步**：提交时自动把当前用户设定发给房主

#### 同步能力

- **聊天历史同步**（支持指定深度，0=全部）
- **流式输出同步**（AI 生成时逐 token 实时推送）
- **正则同步**（character scope）
- **MVU 变量同步**（Generation Ended 后自动推送 stat_data/display_data/delta_data）
- **数据库同步**（支持 AutoCardUpdater 隔离表，房主改表后 3 秒内全员增量/全量同步）
- **剧透遮罩**：消息中 `||内容||` 自动渲染为可点击遮罩

#### 其他实用功能

- 房间内聊天（带时间戳、系统/错误/AI 消息区分）
- 心跳检测 + 自动重连（1s→2s→4s，最多 3 次）
- 拖拽面板 + 移动端完美适配（指针事件 + 软键盘智能判断，不误触发最小化）
- 设置弹窗全覆盖 + 实时预览
- 房间日志持久化（localStorage，分房间保存，最多 150 条）
- 面板加载成功后自动 toastr 提示

### 三、安装方法

1. 确保已安装 [**JS-Slash-Runner**](https://github.com/N0VI028/JS-Slash-Runner)（或同类可执行 JSON 脚本的扩展）
2. 下载本插件文件 `酒馆助手脚本-联机脚本 v0.7.json`
3. 在 JS-Slash-Runner 界面点击 **导入**（Import from JSON）
4. 选择 **全局脚本库**（推荐）或其他分类
5. 保存后打开**全局脚本总开关**，再打开**联机脚本 v0.7** 开关
6. 刷新 ST 页面 → 右上角出现 **“联机工具”** 悬浮面板（绿色状态灯表示待连接）
**注意**：首次加载会自动注入样式和事件监听，热重载后面板会自动重建，不会丢失设置。

### 四、使用教程

#### 1. 本地模式（最简单，推荐新手）
1. 面板中关闭 **“在线模式”**
2. 输入**用户名**（必填）
3. 输入**端口**（默认 2157，所有人必须完全一致）
4. 可选输入密码
5. 点击 **创建房间**（成为房主）或 **加入房间**（成为玩家）
6. 成功后标题栏状态灯变绿

#### 2. 在线模式（多人异地）
1. 打开**设置** → 开启 **“在线模式”**
2. 默认服务器 `https://room.yufugemini.cloud`（可改）
3. 点击右上角刷新按钮加载房间列表
4. 选中房间 → 输入密码 → 点击 **加入**（玩家）或 **观看**（观众）
5. 或直接输入房间名 + 密码 → **创建并加入**

#### 3. 角色与权限
- **房主**：面板显示“你是房主”，拥有“立即发送”“重置”“限时设置”“转让房主”权限
- **玩家**：可提交/撤回输入、聊天、请求同步
- **观众**：面板右下角开关 **观众模式**，开启后无法提交，仅观看（房主强制不可开启）

#### 4. 房主核心玩法
1. 等待所有人提交 → 面板显示“本轮输入池 X/Y ✓ 全部到齐”
2. 点击 **立即发送**（或等待限时自动发送）
3. AI 生成时所有人实时看到流式输出
4. 生成结束后自动同步 MVU / ACU / 正则

#### 5. 玩家日常操作
- “本轮输入”文本框输入内容 → **提交输入**
- 已提交后可 **撤回输入** 再改
- 聊天区随时发消息（Enter 发送）
- 非房主可点击同步历史/正则/变量按钮

#### 6. 界面元素速览
- **状态灯**：绿=已连接，黄=连接中，灰=断开
- **玩家列表**：房主金色皇冠 + 提交后头像变绿 ✓
- **观众列表**：蓝色胶囊
- **转让房主**：房主点击玩家旁边的交换按钮
- **聊天日志**：支持系统/聊天/错误/AI 不同颜色
- **输入池**：隐藏模式下他人内容显示 `********`

### 七、常见问题 FAQ

**Q：为什么别人加不进来？**  
A：在线模式检查服务器地址；本地模式检查端口完全一致 + 同一浏览器环境。

**Q：房主离开后房间会消失吗？**  
A：不会，自动推选最早加入的玩家成为新房主。

**Q：如何同步 ACU 数据库？**  
A：房主把“变量模式”设为“数据库”，修改任意表后自动同步。

**Q：观众能看到输入池吗？**  
A：能看到数量和脱敏内容，但无法提交。

**Q：面板挡住聊天怎么办？**  
A：拖拽到任意位置，最小化后只剩标题栏。

**Q：热重载后设置不见了？**  
A：不会，所有设置、UID、房间日志都持久化在 localStorage。

**Q：为什么有时重连失败？**  
A：手动点击断开按钮会清除重连上下文，需重新连接。

**Q：移动端好用吗？**  
A：已针对触屏、软键盘、视觉视口做了完整适配，不会误最小化。
