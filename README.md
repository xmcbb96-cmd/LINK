### 一、插件简介

**联机脚本 v0.7** 是基于**JS-Slash-Runner**（以下简称 酒馆助手），专为 **SillyTavern**（以下简称 ST）开发的**多人实时协作插件**，支持**在线公共服务器**与**本地频道**两种模式。  
多名用户（玩家 + 观众）可以同时在一个房间内：

- 共同向 AI 提交回复
- 房主统一合并后发送给 AI
- 实时同步聊天记录、流式输出、正则、MVU 变量、数据库
- 支持**隐藏模式**（他人看不到你的具体输入，仅显示 ********）
- 支持**观众模式**（只看不参与提交）

插件以**悬浮毛玻璃面板**形式呈现，可拖拽、最小化、设置，完美融入多种 ST 美化。

---

### 二、核心特性

#### 基础功能
- 在线模式：连接公共服务器（默认 `https://room.yufugemini.cloud`）
- 本地模式：同一浏览器（或同局域网多标签）通过 BroadcastChannel 通信（端口可自定义）
- 创建/加入带密码房间
- 在线房间列表实时刷新 + 人数/锁图标显示
- 玩家 / 房主 / 观众 三种角色动态切换
- 房主自动转让（原房主离开后自动推选最早加入的玩家）

#### 输入协作系统
- **输入池**：所有玩家提交的回复实时显示在面板（房主可见全部，普通玩家可见自己 + 他人脱敏内容）
- **隐藏模式**：开启后他人看到你的输入为 `********`，仅房主能看到真实内容（重 roll 时自动补回上下文）
- **限时自动发送**（房主专属）：设置秒数后有人提交即开始倒计时，超时自动合并发送
- 一键撤回自己已提交的输入

#### 同步能力
- **聊天历史同步**（支持指定深度，防止新加入用户从头开始）
- **流式输出同步**（AI 生成时实时逐 token 推送到所有客户端）
- **正则同步**（character scope）
- **MVU 变量同步**（Generation Ended 后自动推送）
- **数据库全量/增量同步**（支持 AutoCardUpdater 隔离表，房主改表后全员秒同步）
- **Persona 同步**（玩家提交时自动把自己的用户设定发给房主）

#### 其他实用功能
- 房间内聊天（带时间戳、系统消息、错误提示）
- 自动重连（3 次指数退避）
- 拖拽悬浮面板 + 移动端适配（指针事件 + 软键盘智能判断）
- 设置弹窗完整覆盖所有参数
- 本地持久化（房间日志、设置、UID）

### 三、安装方法

1. 确保已安装 [**JS-Slash-Runner** 扩展](https://github.com/N0VI028/JS-Slash-Runner)
2. 下载 酒馆助手脚本-联机脚本 v0.xx.json
3. 在扩展界面点击 **导入**，导入联机工具，选择**全局脚本库**或其他喜欢的位置
4. 打开**全局脚本开关**，然后打开**联机工具**

### 四、使用教程

#### 1. 本地模式（最简单，推荐新手）
1. 打开面板 → 关闭 **“在线模式”**
2. 输入**用户名**（必填）
3. 输入**端口**（默认 2157，同一房间所有人必须一致）
4. 可选输入密码
5. **创建房间**（房主） 或 **加入房间**（玩家）
6. 成功后面板标题变为绿色状态灯

#### 2. 在线模式（多人异地联机）
1. 面板中打开**设置** → 开启 **“在线模式”**
2. 默认服务器为 `https://room.yufugemini.cloud`（可自行更换）
3. 点击 **刷新** 按钮加载房间列表
4. 选中房间 → 输入密码（如果有）→ 点击 **加入** 或 **观看**
5. 或者直接 **创建并加入** 新房间

#### 3. 角色切换
- **房主**：面板显示“你是房主”，拥有合并发送、重置、限时设置等权限
- **玩家**：可提交输入、撤回、聊天
- **观众**：面板右下角开关 **观众模式**，开启后无法提交输入，仅观看（房主不可切换为观众）

#### 4. 房主操作流程（核心玩法）
1. 所有人提交输入 → 面板显示“本轮输入池 X/Y”
2. 全部到齐后点击 **立即发送**（或等待限时自动发送）
3. AI 生成时所有人实时看到流式输出
4. 生成结束后自动同步 MVU / 数据库 / 正则

#### 5. 玩家操作
- 在“本轮输入”文本框输入内容 → 点击 **提交输入**
- 想修改可点击 **撤回输入** 后再重新提交
- 聊天区随时发消息

### 五、设置详解（点击面板右上角 ⚙️）

| 设置项                  | 说明                                                                 | 默认值          | 备注 |
|-------------------------|----------------------------------------------------------------------|-----------------|------|
| **在线模式**            | 切换在线/本地模式                                                    | 开启            | 影响房间列表与服务器地址 |
| **观众模式**            | 切换为只观看不提交                                                   | 关闭            | 房主强制关闭 |
| **服务器地址**          | 在线模式的后端地址                                                   | room.yufugemini.cloud | 可自建 |
| **用户名**              | 房间内显示名称（实时广播）                                           | -               | 支持 {name} 占位符 |
| **UID**                 | 用户唯一标识（断开后可改）                                           | 随机生成        | 用于绑定身份 |
| **变量模式**            | 房主设置：none / mvu / apotheosis                                    | none            | 影响同步内容 |
| **隐藏模式**            | 他人看不到你的输入具体内容                                           | 关闭            | 仅房主可见真实内容 |
| **限时输入 (秒)**       | 房主专属：有人提交后倒计时自动发送                                   | 0（关闭）       | 推荐 30-60 秒 |
| **同步历史消息层数**    | 新加入用户一次性拉取的历史消息数量                                   | 10              | 0 = 全部 |
| **发送用户设定**        | 提交输入时自动把自己的 Persona 发给房主                              | 开启            | 强烈推荐 |
| **设定前缀**            | Persona 前缀模板                                                     | `[{name}]的设定:` | 支持 {name} |
| **消息前缀**            | 每条输入的前缀模板                                                   | `[{name}]:`     | 支持 {name} |
| **消息后缀**            | 每条输入的后缀（desu!! 等）                                          | 空              | - |

**预览框** 会实时显示当前前缀 + 后缀 + Persona 的效果。

### 六、高级功能深度说明

#### 1. 隐藏模式 + 重 roll 完美支持
开启隐藏模式后：
- 普通玩家提交的内容在他人面板显示 `********`
- 重 roll 时，插件会自动把隐藏内容作为系统提示补回，确保生成一致

#### 2. 数据库同步
- 房主修改任何 数据库 隔离表后，**3 秒内** 全员自动增量/全量同步
- 无需手动导出导入，真正实现“多人共用同一张卡”

#### 3. 流式输出
AI 生成时，所有客户端实时逐 token 出现，与单人使用体验一致。

#### 4. 自动重连
掉线后自动 1s → 2s → 4s 尝试重连，最多 3 次，成功后自动恢复房间状态。

### 七、常见问题 FAQ

**Q：为什么我创建房间后别人加不进来？**  
A：在线模式请确认服务器地址一致；本地模式请确认端口完全相同，且在**同一浏览器**（或支持 BroadcastChannel 的环境）。

**Q：房主离开后房间会解散吗？**  
A：不会！会自动把最早加入的玩家升为新房主。

**Q：如何同步 数据库 表？**  
A：房主开启“变量模式 → 数据库”，修改任意表后自动同步；新加入用户也会自动收到全量数据。

**Q：观众能看到输入池吗？**  
A：能看到，但无法提交，且隐藏模式下仍显示脱敏内容。

**Q：面板挡住了聊天区怎么办？**  
A：拖拽到左上角或右上角，最小化后只剩标题栏。

**Q：热重载后设置丢失？**  
A：所有设置（包括 UID、日志）都持久化在 localStorage，不会丢失。
